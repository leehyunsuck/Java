# Array

여러 개의 값을 하나의 변수에 저장하기 위해 사용되는 데이터 구조 <br>
<small> - 배열의 크기 고정<br> - 인덱스로 접근 </small>

### 일차원 배열

`데이터타입[] 배열명 = new 데이터타입[크기]`로 배열 선언<br>
`배열명[인덱스] = 값`으로 인덱스위치에 값 설정<br>

또는 `데이터타입[] 배열명 = { 값, 값, ... };`

`배열명[인덱스]` 로 값 꺼내기
```java
class Main {
    public static void main(String[] args) {
        int[] ints = new int[5];
        ints[0] = 1;
        ints[1] = 2;
        // ...
        System.out.println(ints[1]); // 2
    }
}
```
### 다차원 배열

`데이터타입[][] 배열명 = new 데이터타입[행의 개수][열의 개수];`
```java
//...
        int[][] ints = new int[2][3];
        /*  
            { 
                {값, 값, 값}
                {값, 값, 값}
            }
        */
//...
```

# Enumeration

관련된 상수값들을 그룹화하여 사용하는 방법

클래스처럼 정의되며, 열거형 상수는 해당 열거형의 유일한 값으로 존재<br>
<small> 힙 영역에 저장됨 </small>

값 비교는 같은 열거형안의 값만 비교 가능<br>

<big> 선언 </big> <br>
`enum 열거형 { 값1, 값2, ... }`<br>

<big> 객체 선언 </big> <br>
`열거형 객체 = 열거형.값;` <br>

<big> 열거형 값 전체 사용 </big> <br>
`열거형.values()`

```java
//...
    enum Gender {
        MALE, FEMALE
    }
    public static void main(String[] args) {
        Gender hyunsuck = Gender.MALE;

        if (hyunsuck == Gender.MALE) {
            System.out.println("남성임");
        }

        Gender genders = Gender.values();
        for (Gender g : genders) {
            System.out.println(g);
        }
    }
//...
```

# Inheritance

이미 정의된 클래스의 필드와 메소드를 새로운 클래스에 재사용 할 수 있음<br>

<small> 하위 클래스 인스턴스 생성시 상위클래스 생성자 + 하위클래스 생성자 실행됨 </small>

- 단일 상속 (다중 상속 불가능)<br>
<small> 한 클래스는 하나의 클래스만 상속 가능 </small>

- 다단계 상속 가능<br>
<small> C가 B상속받고, B가 A상속받는 형식 가능 </small>

- 인터페이스 상속 <br>
<small> 클래스만 인터페이스 상속 받을 수 있음 <br>
클래스는 인터페이스 여러개 상속받기 가능 </small><br><br>

class 클래스 <big>`extends`</big> 상위클래스 { ... } <br>
class 클래스 <big>`implements`</big> 인터페이스 { ... }<br>

### IS-A (상속관계)
하위클래스에 상위클래스 상속 <br>
`하위클래스 extends 상위클래스;`<br>

Pizza is a Food<br>
<small> 피자 클래스가 음식 클래스를 상속 받음 </small>

```java
class Food {
    public Food() {
        System.out.println("Food 생성자 실행");
    }
    public void whatIsThat() {
        System.out.println("음식이야!");
    }
}

class Pizza extends Food {
    public Pizza() {
        System.out.println("Pizza 생성자 실행");
    }
    public void eat() {
        System.out.println("피자 냠냠");
    }
}

public class Main {
    public static void main(String[] args) {
        Pizza pizza = new Pizza();
            // Food 생성자 실행
            // Pizza 생성자 실행
        pizza.whatIsThat();
            // 음식이야!
        pizza.eat();
            // 피자 냠냠
    }
}
```

### HAS-A (소유관계)
상위 클래스에 필드 선언 <br>
`하위클래스 필드명;`<br>

Aggregation : 별도로 만들어서 끼워넣음 (라이프 사이클 다름)<br>
Composition : 객체 생성과 동시에 생성 (라이플 사이클 같음)

Pizza has a Cheese<br>
<small> 피자 클래스가 치즈 클래스를 갖고있음 </small>

```java
//Composition 예제 코드
class Cheese {
    public void topping() { 
        System.out.println("치즈 뿌리기");
    }
}

class Pizza {
    Cheese cheese;

    public Pizza() {
        this.cheese = new Cheese();
    }

    public void cook() {
        cheese.topping();
        System.out.println("오븐에 굽기 시작");
    }
}

public class Main {
    public static void main(String[] args) {
        Pizza pizza = new Pizza();
        pizza.cook();
            // 치즈 뿌리기
            // 오븐에 굽기 시작
    }
}
```

### USE-A (의존관계)
Oven use a electricity
<small> 오븐은 전기를 사용한다 </small>

```java
interface electricity {
    public void useElectricity();
}

class Oven implements electricity {
    @Override
    public void useElectricity() {
        System.out.println("오븐은 전기를 사용");
    }
}

public class Main {
    public static void main(String[] args) {
        Oven oven = new Oven();
        Oven.useElectricity();
    }
}
```
